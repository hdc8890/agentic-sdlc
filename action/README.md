# Agentic SDLC GitHub Action

The Agentic SDLC Action integrates your repository with the Agentic SDLC framework via CI — no global CLI install needed.

## Usage

### Validate on every push

Add this workflow to validate `.agentic/` configuration on every change:

```yaml
# .github/workflows/agentic-validate.yml
name: Agentic SDLC Validate

on:
  push:
    paths:
      - '.agentic/**'
  pull_request:
    paths:
      - '.agentic/**'

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: your-org/agentic-sdlc@v1
        with:
          command: validate
```

### Onboard a new repository

Run once to generate `.agentic/` and open a PR for review:

```yaml
# .github/workflows/agentic-onboard.yml
name: Agentic SDLC Onboard

on:
  workflow_dispatch:    # Trigger manually for first-time onboarding

jobs:
  onboard:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: your-org/agentic-sdlc@v1
        with:
          command: init
          autonomy_level: semi_autonomous

      - uses: your-org/agentic-sdlc@v1
        with:
          command: profile-generate

      - name: Open PR with generated config
        uses: peter-evans/create-pull-request@v6
        with:
          title: 'chore: onboard to Agentic SDLC framework'
          body: |
            This PR adds `.agentic/` configuration generated by the Agentic SDLC CLI.

            **Review checklist:**
            - [ ] Confirm `profile.json` has the correct test/lint/build commands
            - [ ] Adjust `policy.json` autonomy level for your team
            - [ ] Add any additional `protected_paths` the tool didn't detect
            - [ ] Set `repository.owner` to your GitHub org name
          branch: 'agentic/onboarding'
          commit-message: 'chore: add .agentic/ onboarding config'
```

### Re-detect repo profile after structural changes

```yaml
# Run manually when you add a new package manager, change CI, etc.
- uses: your-org/agentic-sdlc@v1
  with:
    command: profile-generate
```

## Inputs

| Input | Required | Default | Description |
|---|---|---|---|
| `command` | Yes | `validate` | `validate`, `profile-generate`, `init`, or `status` |
| `autonomy_level` | No | `semi_autonomous` | For `init` command only |
| `working_directory` | No | `.` | Repository root |

## Outputs

| Output | Description |
|---|---|
| `profile_path` | Path to `.agentic/profile.json` (after `init` or `profile-generate`) |
| `validation_passed` | Whether validation passed (after `validate`) |

## What the action does

1. Installs the `@agentic-sdlc/cli` package
2. Runs the requested command in the working directory
3. On validation failure, exits non-zero (blocks the PR/push)
4. On profile generation, writes `.agentic/profile.json` for the PR step to commit
